# 材料文件名 = 材料名称、单目录、按文件夹组织 — 设计

日期：2026-06-18

## 背景与目标

当前材料（材料 / Material）的存储方式：

- 物理路径：`{dataRoot}/{safeDir(售后单号)}/images|videos/{原始文件名}`
- 逻辑文件夹 `Material.folder`（如 `凭证/聊天`）**仅是数据库里的字符串标签**，用于 UI 分组，不影响磁盘布局。
- `Material.name` 是独立的显示名，与磁盘文件名无关。
- 同名冲突通过给文件名追加 `-1`、`-2` 后缀解决。

用户要求：

1. **新建材料时，把磁盘文件名重命名为「材料名称」，后缀不变。**
2. **所有材料放在一个目录里，不再区分 `videos`/`photos`。**
3. **材料名称在同一目录下唯一。**

经澄清，用户的心智模型是「物理目录 = 逻辑文件夹」。因此本设计让**物理布局镜像逻辑文件夹树**，使「同一目录下唯一」自然成立（每个文件夹就是一个物理目录，文件名 = 材料名 + 后缀，故不会物理冲突）。

## 核心模型变化

物理布局改为镜像逻辑文件夹树，文件名 = 经过校验的材料名 + 原始后缀：

```
之前： {dataRoot}/{售后单号}/images/原始文件名.jpg     (folder 仅是数据库标签)
       {dataRoot}/{售后单号}/videos/原始视频名.mp4
之后： {dataRoot}/{售后单号}/凭证/聊天/客服对话.jpg     (图片与视频同目录)
       {dataRoot}/{售后单号}/凭证/物流/快递单.mp4
       {dataRoot}/{售后单号}/根目录材料.png            (根文件夹 = 直接放在售后单目录下)
```

- `kind`（image/video）保留，作为缩略图生成与 UI 筛选的元数据，但**不再决定目录**。
- `name` 成为文件名主干（不再与磁盘名分离）。

## 校验与唯一性（在主进程强制执行——以主进程为准）

- **名称必填**：空 / 全空白 → 抛 `请输入材料名称`。
- **拒绝非法字符**：名称含 `/ \ : * ? " < > |`、控制字符，或为 `.`/`..`/以点结尾 → 抛 `材料名称不能包含 / \ : * ? " < > | 等字符`。
- **(售后单, folder) 内唯一**：该文件夹下已存在同名材料 → 抛 `该文件夹下已存在同名材料`。在**新建**与**移动到某文件夹**时检查。

## 新增 / 修改的辅助函数（`src/main/services/paths.ts`）

- `safeSeg(s)` — 清洗单个路径段（复用 `safeDir` 的逻辑），用于防御性地构造物理文件夹目录（处理历史遗留的非法文件夹名）。
- `assertValidMaterialName(name)` — 校验失败时抛出上述错误。
- `materialDir(dataRoot, no, folder)` / `materialRelPath(no, folder, name, ext)` — 路径构造的唯一来源，供新建、移动、文件夹重命名共用，使不变式集中在一处。
- `MaterialRepo.nameTaken(aftersaleNo, folder, name, exceptId?)` — 唯一性查询。

## 各操作行为（「全部同步」）

1. **新建**（`Importer.addFile` / `addBytes`）：校验名称 → 唯一性检查 → 写入 `materialDir/...{name}{ext}`（去掉旧的 `images|videos` 拆分与 `-1` 后缀去重；唯一性已保证不冲突）→ 记录入库。
2. **移动到文件夹**（`materials:move`）：在目标文件夹做唯一性检查 → 在磁盘 `rename()` 文件到新文件夹目录 → 更新数据库 `rel_path` + `folder`。（缩略图文件不移动，`thumb_path` 仍有效。）
3. **重命名 / 移动文件夹**（`folders:rename`）：`FolderRepo.rename` 现已重写数据库里的 `folder` 与子文件夹路径；额外计算每个受影响材料的新 `rel_path`（= 新 folder + name + ext）并更新，返回 `{oldRelPath,newRelPath}[]`；IPC 层据此在磁盘上 `rename` 文件（与 `folders:remove` 依据返回列表删除文件的既有模式一致）。
4. **删除文件夹**（`folders:remove`）：不变（已按 `relPath` 删除文件）；额外尽力 `rmdir` 清空后的文件夹目录。
5. **删除售后单 / 导出 / 扫描校准**：不变——它们基于 `relPath`/`folder`，仍正确。（附带收益：导出的文件名现在反映真实材料名。）

## 历史遗留材料（只影响新建，但保持一致）

旧材料仍留在 `images/`/`videos/`、使用原始文件名，**不做前置迁移**。若某次文件夹重命名或移动**触及**到它，则它被迁移到新布局（文件名 = name+ext；若 `name` 为空则保留其当前 basename）。未被触及的旧材料通过其存储的 `relPath` 继续正常工作。

## 文件系统与数据库一致性

FS 与 DB 无法在同一事务内提交。采用务实策略：先计算变更计划 → 执行 FS 移动 → 更新 DB；FS 失败则在改动 DB 前中止，并尽力回滚已移动的文件。对单用户桌面应用，这种尽力而为并把错误抛给用户的方式可接受。

## 渲染层（`NewMaterialDialog.tsx`）

- **名称改为必填**：名称为空 / 全空白时禁用「创建」按钮。
- 透出抛出的错误（已接入错误横幅）——重复名 / 非法字符的提示就地显示。

## 不在本次范围

- 不做历史数据的前置迁移。
- 不改缩略图命名。
- 导出 / zip 逻辑除自然受益外不额外改动。

## 已确认的决策

- 唯一性范围：每个逻辑文件夹内唯一（= 其物理目录）。
- 重名冲突：报错阻止，要求改名。
- 名称非法字符：报错阻止。
- 文件夹相关操作（移动材料、重命名 / 移动文件夹）全部同步磁盘文件。
- 根文件夹 = 文件直接放在售后单目录下。
- 移除仅测试使用的 `importFiles`。
